Playwright tests, written in English.

Describe a UI scenario in plain English. Promptomate clicks through your site, writes a real Playwright spec, and self-heals when the UI changes. CI tells you which red tests are real bugs.

Everything Playwright, plus the AI it was missing.

🧭

Agent-driven exploration

Claude drives a real Chromium through Microsoft's Playwright MCP — navigate, click, hover, drag, fill, tab, network. It figures out multi-step flows, then writes the spec.

🩹

Self-healing locators

Button renamed? Form restructured? One command re-resolves drifted locators against the current DOM. Or run triage --apply in CI to heal automatically.

🔎

AI failure triage

When a test fails in CI, Claude classifies it: real bug / flake / DOM drift. The PR comment tells your team which red tests actually block the merge.

👁️

Visual assertions

"Verify a red error banner appears" — no DOM selector, just a description. Claude Vision judges the screenshot and throws if it doesn't match.

🔐

Secrets safely

${'${VARNAME}'} in your prompt becomes process.env.VARNAME in the generated spec — passwords never land in committed files.

🔑

Auth fixtures

Log in once, reuse the session across tests. CI runs the auth fixture first, then fans out the rest in parallel.
